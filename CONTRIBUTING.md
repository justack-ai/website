## Developer Certificate of Origin (DCO)

All contributions must include a DCO sign-off. This certifies that you have the right to submit your contribution under the project's open source license.

### How to sign off

Add a `Signed-off-by` line to every commit message:

```
Signed-off-by: Your Name <your.email@example.com>
```

The easiest way is to use the `-s` flag when committing:

```bash
git commit -s -m "Description of your change"
```

Your sign-off must use your real name (not a pseudonym) and a valid email address.

### What the DCO means

By signing off, you certify the statements in the [Developer Certificate of Origin](DCO). In short: you have the right to submit the code, and you understand it will be distributed under the project's Apache 2.0 license.

## Contribution process

1. Fork the repository
2. Create a feature branch from `main`
3. Make your changes
4. Ensure all commits are signed off (`git commit -s`)
5. Submit a pull request

## Code of conduct

Be respectful, constructive, and mindful that this project serves people navigating the justice system. Contributions that undermine access to justice are not welcome.

## Responsible use

This project includes a [Responsible Use Addendum](RESPONSIBLE-USE.md) that prohibits certain uses of the software. Please review it before contributing.
